# Internship_Case_Study_Day14

The first machine learning model was implemented for the fraud detection system. Logistic Regression was selected as the baseline model because it is simple, interpretable, and commonly used for binary classification problems.

The purpose of this model is to provide a benchmark for comparing more advanced models such as Random Forest and XGBoost.


Objectives
The objectives of this step were:
- Train a Logistic Regression model on the prepared dataset
- Predict fraud and legitimate transactions
- Evaluate model performance using classification metrics
- Establish a baseline for comparison with other models


Model Training Process
The following steps were performed:
1. Load the preprocessed dataset
2. Train the Logistic Regression model
3. Generate predictions on the test dataset
4. Evaluate model performance


Evaluation Metrics
The following metrics were used to evaluate the model:
- Accuracy
- Precision
- Recall
- F1 Score
- Confusion Matrix
Since fraud detection datasets are imbalanced, special attention is given to **Recall**, which measures how many fraudulent transactions are correctly detected.

Expected Outcome
The Logistic Regression model will serve as a baseline reference for evaluating more advanced models.
Performance results from this model will be compared with:
- Random Forest
- XGBoost
to determine which model performs best for fraud detection.
